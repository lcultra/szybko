# Szybko 愿景

Szybko 是一个本地优先的桌面生产力入口。它从快速启动器开始，但长期形态是一个插件化平台：承载搜索、指令、本地自动化、上下文工具和 AI 辅助工作流。

产品体验应该足够直接：唤起 Shell，输入意图，选择动作，然后回到当前工作。插件扩展的是这个流程，而不是把用户带进一组割裂的应用。

## 产品原则

- 本地优先。核心搜索、指令执行、插件运行时和用户数据不应依赖网络才能工作。
- 意图发生点要快。Shell 必须快速出现、立即接受输入，并尽早流式返回有用结果。
- 插件是一等能力。插件不只是搜索来源，也可以提供指令、UI、持久状态、系统能力和长生命周期会话。
- 展示位置变化不应破坏运行时状态。插件从主 Shell 移到浮动窗口，或未来移动到侧边栏、分屏等宿主时，不应默认重建会话。
- 原生能力必须被平台中介。文件、剪贴板、进程、窗口、图像和自动化能力应通过适配器与权限边界提供，而不是散落在插件实现里。
- 兼容要务实。uTools 风格的插件概念有价值，但 Szybko 不应为了完全兼容而牺牲自身架构。

## 长期用户体验

Szybko 支持三类连续工作流：

- 搜索：查找文件、应用、指令、插件功能、最近动作和上下文结果。
- 执行：直接从结果触发动作，例如打开资源、转换文本、启动插件功能或调用原生能力。
- 延续：当插件需要状态时，让会话持续存在，例如聊天、浏览器式工具、偏好设置或多步骤操作。

Launcher 是主要入口，但不是唯一展示面。一个插件运行时可以显示在 Launcher 中，也可以分离到浮动窗口，回到 Launcher，或未来挂载到侧边栏、分屏和停靠区域。

## 平台范围

平台长期应包含：

- 插件目录：发现已安装插件及其声明的能力。
- 指令目录：索引插件指令、别名、动态功能和用户偏好。
- 运行时系统：创建、跟踪、挂起、恢复和销毁插件会话。
- 宿主系统：在不同展示面中承载运行时。
- 原生适配器层：通过稳定的领域概念暴露操作系统能力。
- 持久化：保存应用设置、插件数据、指令偏好和市场元数据。
- 权限与沙箱：为第三方插件提供可控的能力边界。

## 蓝图定位

蓝图描述产品和平台的长期方向。它刻意不包含类名、方法签名、IPC 载荷、文件路径、框架绑定或当前迁移步骤。

阶段性实现决策属于 dated specs 和 plans。蓝图应该在当前代码结构变化后依然有效。
